# GUI界面使用指南

## 📋 概述

Java项目分析智能体提供直观的图形界面，帮助用户轻松分析Java项目并进行交互式学习。

---

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动应用

```bash
python run.py
```

或直接双击 `run.py` 文件。

## 📖 功能说明

### 1. 选择项目

**操作步骤：**
1. 点击 **"📁 选择项目"** 按钮
2. 在弹出的对话框中选择Java项目根目录
3. 系统会自动验证是否为有效的Java项目（检查 `src/main/java` 目录）

**要求：**
- 必须是标准的Maven或Gradle项目
- 包含 `src/main/java` 目录
- 建议包含 `pom.xml` 或 `build.gradle`

---

### 2. 模型配置

**操作步骤：**

#### 方式一：快速切换（推荐）
1. 在工具栏的 **"模型配置"** 下拉框中选择已保存的配置
2. 系统自动加载该配置的所有参数
3. 配置会自动保存到使用记录中

#### 方式二：管理配置
1. 点击 **"⚙️ 管理配置"** 按钮
2. 在弹出的配置管理对话框中：

**左侧 - 配置列表：**
- 显示所有已保存的配置
- 默认配置会标注 "⭐ (默认)"
- 支持编辑、设为默认、删除操作

**右侧 - 编辑区：**
- 填写配置信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **配置名称** | 自定义名称（必填） | `OpenAI GPT-4` |
| **API Key** | LLM API密钥（必填） | `sk-xxxxxxxxxxxx` |
| **Base URL** | API基础URL（可选） | `http://localhost:11434/v1` |
| **模型名称** | 使用的模型 | `gpt-4`, `gpt-3.5-turbo` |

3. 点击 **"💾 保存配置"** 按钮

**配置文件位置：**
```
klgm1/config/model_configs.json
```

**常用配置模板：**

#### OpenAI官方
```
配置名称: OpenAI GPT-4
API Key: sk-xxx
Base URL: (留空)
模型名称: gpt-4
```

#### Ollama本地
```
配置名称: Ollama Local
API Key: ollama
Base URL: http://localhost:11434/v1
模型名称: llama2
```

#### DeepSeek
```
配置名称: DeepSeek
API Key: sk-xxx
Base URL: https://api.deepseek.com/v1
模型名称: deepseek-chat
```

#### Azure OpenAI
```
配置名称: Azure GPT-4
API Key: xxx
Base URL: https://your-resource.openai.azure.com/openai/deployments/your-deployment
模型名称: gpt-4
```

**配置管理功能：**

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📝 编辑 | 加载配置到编辑区 | 选择配置后点击，可修改参数 |
| ⭐ 设为默认 | 设置默认配置 | 程序启动时自动加载 |
| 🗑️ 删除 | 删除配置 | 确认后永久删除 |
| 💾 保存配置 | 保存新配置或更新 | 保存后立即可用 |

**配置使用统计：**
- 系统自动记录每个配置的使用次数
- 记录最后使用时间
- 常用配置会显示在列表顶部

**默认配置机制：**
1. 首次启动时自动创建3个默认配置
2. 程序启动时自动加载默认配置
3. 可随时更改默认配置
4. 默认配置会标注 "⭐ (默认)"

---

### 3. 分析项目

**操作步骤：**
1. 确保已选择项目并配置API密钥
2. 点击 **"🔍 分析项目"** 按钮
3. 等待分析完成（通常需要10-30秒）
4. 查看生成的 `overview.md` 内容

**分析内容：**
- 项目框架及技术栈分析
- 学习路线图
- 已学习内容（初始为空）
- 项目代码索引

**状态提示：**
- ⏳ 正在分析项目...
- ✅ 项目分析完成
- ❌ 分析失败: [错误信息]

---

### 4. 学习知识点

**操作步骤：**
1. 项目分析完成后，**"📚 下一个知识点"** 按钮会启用
2. 点击该按钮开始学习第一个知识点
3. 系统会：
   - 生成知识点讲解（保存到 `note/{topic}.md`）
   - 更新 `overview.md` 中的学习进度
   - 自动切换到文档查看Tab显示内容

**学习内容结构：**
```
# 知识点标题

> 🎯 学习目标: ...

---

## 一、这是什么？为什么重要？
...

## 二、核心原理大白话
...

## 三、项目实战演练
...

## 四、常见陷阱与解决方案
...

## 五、进阶技巧
...

## 六、知识扩展
...

## 七、小结与下一步
...
```

---

### 5. 问答互动

**操作步骤：**
1. 在学习知识点后，输入框和提问按钮会启用
2. 在输入框中输入问题
3. 按 **Enter键** 或点击 **"💬 提问"** 按钮
4. 等待AI回答（通常需要2-5秒）
5. 自动切换到"问答记录"Tab查看对话历史

**示例问题：**
- "@SpringBootApplication注解的作用是什么？"
- "这个知识点在项目哪里用到了？"
- "能给我一个更简单的例子吗？"
- "常见的错误有哪些？"

**对话历史：**
- 每次问答都会记录在"问答记录"Tab中
- 切换知识点时会清除对话历史
- 支持滚动查看完整对话

---

### 6. 查看学习进度

**操作步骤：**
1. 点击 **"📊 学习进度"** Tab
2. 查看以下信息：
   - 已完成知识点数量
   - 进度百分比
   - 进度条可视化
   - 已学习的知识点列表
   - 下一个建议学习的知识点

**进度更新：**
- 每完成一个知识点自动更新
- 包含掌握程度评估（1-5星）
- 时间戳记录

---

### 7. 查看Token消耗

**操作步骤：**
1. 点击 **"💰 Token消耗"** Tab
2. 查看以下信息：
   - 总调用次数
   - 总Token消耗量
   - 每次调用的详细记录表格
   - 按操作类型分类统计

**表格字段说明：**

| 字段 | 说明 |
|------|------|
| **时间** | 调用发生的时间 |
| **操作** | 操作类型（项目分析/知识点讲解/问答/进度更新） |
| **模型** | 使用的模型名称 |
| **输入Token** | 发送给模型的token数量 |
| **输出Token** | 模型返回的token数量 |
| **总Token** | 本次调用的总token消耗 |

**功能按钮：**

- **📄 生成Token报告**：生成详细的Markdown格式报告，包含统计信息和完整记录
- **🗑️ 清空记录**：清空所有token消耗记录（谨慎使用）

**Token消耗参考：**

| 操作 | 预估Token | GPT-4成本 | GPT-3.5成本 |
|------|-----------|-----------|-------------|
| 项目分析 | ~5000-8000 | $0.15-$0.24 | $0.01-$0.016 |
| 知识点讲解 | ~4000-6000 | $0.12-$0.18 | $0.008-$0.012 |
| 回答问题 | ~1500-3000 | $0.045-$0.09 | $0.003-$0.006 |
| 进度更新 | ~2000-3000 | $0.06-$0.09 | $0.004-$0.006 |

**省钱技巧：**
- 使用Token消耗Tab监控实际使用情况
- 对简单任务使用GPT-3.5-turbo
- 避免重复分析同一项目
- 定期查看报告，优化使用策略

---

### 8. 重置状态

**操作步骤：**
1. 点击 **"🔄 重置"** 按钮
2. 系统会：
   - 清空所有显示内容
   - 重置按钮状态
   - 清除聊天历史
   - 重置进度显示

**注意：**
- 不会删除已生成的文件（`overview.md`, `note/*.md`）
- 仅重置UI状态

## 💡 使用技巧

### 1. 分阶段使用不同模型

**推荐策略：**
```
项目分析: GPT-4 (质量优先)
知识点讲解: GPT-3.5-turbo (成本优化)
问答互动: GPT-3.5-turbo (速度快)
```

**操作方法：**
1. 分析项目前配置为 `gpt-4`
2. 分析完成后，打开模型配置改为 `gpt-3.5-turbo`
3. 继续学习知识点

**预计节省：** 60% API费用

---

### 2. 高效学习流程

```
1. 选择项目 → 2. 分析项目 → 3. 阅读overview.md
                                    ↓
4. 学习知识点 → 5. 阅读note文件 → 6. 提问互动
                                    ↓
7. 查看进度 → 8. 监控Token消耗 → 9. 学习下一个知识点 (回到步骤4)
```

**新增：Token监控**
- 每次LLM调用后自动记录Token消耗
- 在"💰 Token消耗"Tab中查看实时统计
- 使用"生成Token报告"功能导出详细报告
- 根据消耗情况优化模型选择策略

---

### 3. 文件管理

**生成的文件：**
```
java-project/
├── overview.md              # 项目分析报告
└── note/                    # 知识点讲解目录
    ├── core_java.md
    ├── spring_boot.md
    └── ...
```

**建议：**
- 定期备份 `note/` 目录
- 可使用Markdown编辑器查看（如VS Code + Markdown插件）
- 支持导出为PDF或HTML

---

### 4. 常见问题

#### Q1: 点击"分析项目"后界面无响应？
**A:** 这是正常现象，分析需要10-30秒。界面会在后台执行，完成后自动更新。如果长时间无响应，检查：
- API密钥是否正确
- 网络连接是否正常
- 项目路径是否有效

#### Q2: 如何更换API密钥？
**A:** 点击"模型配置"按钮，修改API Key字段后保存即可。

#### Q3: 能否同时分析多个项目？
**A:** 当前版本不支持。如需切换项目：
1. 点击"重置"按钮
2. 重新选择项目
3. 重新分析

#### Q4: 问答历史记录在哪里？
**A:** 切换到"💬 问答记录"Tab即可查看。切换知识点时会清空历史。

#### Q5: 如何查看之前学习的知识点？
**A:** 
- 方法1: 直接打开 `note/` 目录下的 `.md` 文件
- 方法2: 在"📄 文档查看"Tab中手动加载文件

## ⚠️ 注意事项

### 1. API费用控制

**估算成本：**
| 操作 | Tokens | GPT-4成本 | GPT-3.5成本 |
|------|--------|-----------|-------------|
| 项目分析 | ~6000 | $0.18 | $0.012 |
| 知识点讲解 | ~5000 | $0.15 | $0.01 |
| 回答问题 | ~2000 | $0.06 | $0.004 |

**省钱技巧：**
- 使用GPT-3.5-turbo进行日常学习
- 仅在复杂分析时使用GPT-4
- 避免重复分析同一项目

---

### 2. 网络要求

- 需要稳定的互联网连接（使用OpenAI API时）
- 建议使用有线网络或稳定的WiFi
- 如遇超时，检查防火墙设置

---

### 3. 系统要求

**最低配置：**
- Python 3.8+
- 2GB RAM
- 100MB磁盘空间

**推荐配置：**
- Python 3.10+
- 4GB RAM
- SSD硬盘

---

### 4. 数据安全

**隐私保护：**
- API密钥仅存储在内存中，关闭应用后消失
- 项目代码不会上传到外部服务器（仅发送给LLM API）
- 建议使用环境变量存储API密钥：
  ```bash
  export OPENAI_API_KEY="sk-xxx"
  ```

## 🔧 故障排除

### 问题1: 导入错误

**错误信息：**
```
ModuleNotFoundError: No module named 'langchain'
```

**解决方案：**
```bash
pip install -r requirements.txt
```

---

### 问题2: API密钥无效

**错误信息：**
```
AuthenticationError: Invalid API key
```

**解决方案：**
1. 检查API密钥是否正确复制（无多余空格）
2. 确认账户余额充足
3. 尝试重新生成API密钥

---

### 问题3: 项目分析失败

**错误信息：**
```
RuntimeError: 项目分析失败: ...
```

**解决方案：**
1. 确认项目路径包含 `src/main/java`
2. 检查是否有 `pom.xml` 或 `build.gradle`
3. 查看 `overview.md` 中的详细错误信息

---

### 问题4: 界面冻结

**症状：**
- 点击按钮后界面无响应
- 无法拖动窗口

**解决方案：**
1. 等待30秒，可能是LLM响应慢
2. 如果持续无响应，强制关闭应用
3. 检查网络连接
4. 尝试降低模型复杂度（改用GPT-3.5）
