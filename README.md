# Java项目分析智能体

一个基于GUI的智能化Java项目学习辅助工具，通过AI代理帮助用户分析和学习代码项目。

---

## 🎯 核心功能

### 1. 智能项目分析
- ✅ 自动分析Java项目结构和技术栈
- ✅ 生成项目概览报告（`overview.md`）
- ✅ 识别学习路线图

### 2. 知识点讲解
- ✅ 按顺序生成知识点讲解
- ✅ 保存笔记到 `note/` 目录
- ✅ 更新学习进度

### 3. AI问答互动
- ✅ 基于当前上下文的智能问答
- ✅ 多轮对话支持
- ✅ 对话历史记录

### 4. Token消耗监控
- ✅ 实时追踪每次LLM调用的Token消耗
- ✅ 按操作类型统计（分析/讲解/问答）
- ✅ 生成详细使用报告
- ✅ 数据持久化（`token_usage.json`）

### 5. 模型配置管理
- ✅ 保存多个模型配置
- ✅ 快速切换不同配置
- ✅ 设置默认配置
- ✅ 自动加载上次使用配置
- ✅ 使用统计（次数、时间）
- ✅ 配置文件持久化（`config/model_configs.json`）

### 6. 学习进度可视化
- ✅ 已完成/总知识点数量
- ✅ 进度百分比和进度条
- ✅ 已学习知识点列表
- ✅ 下一个建议学习

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置模型

#### 方式一：GUI配置（推荐）
1. 启动应用：`python run.py`
2. 点击 "⚙️ 管理配置"
3. 添加你的API Key和模型信息
4. 点击 "💾 保存配置"

#### 方式二：环境变量
```bash
# Windows
set OPENAI_API_KEY=sk-your-key

# Linux/Mac
export OPENAI_API_KEY="sk-your-key"
```

### 3. 启动应用

```bash
python run.py
```

### 4. 使用流程

```
1. 选择项目 → 2. 分析项目 → 3. 阅读overview.md
                                    ↓
4. 学习知识点 → 5. 阅读note文件 → 6. 提问互动
                                    ↓
7. 查看进度 → 8. 学习下一个知识点 (回到步骤4)
```

## 📦 技术架构

### 目录结构

```
klgm1/
├── config/
│   ├── knowledge_tree.json        # 知识点树配置
│   └── model_configs.json         # 模型配置文件
├── core/
│   ├── agent_manager.py           # LLM管理器
│   ├── context_manager.py         # 上下文管理器
│   ├── orchestrator.py            # 业务流程编排器
│   ├── token_tracker.py           # Token消耗追踪器
│   ├── model_config_manager.py    # 模型配置管理器
│   └── prompts/                   # 提示词模板
├── gui/
│   └── main_window.py             # GUI主窗口
├── tests/
│   ├── test_token_tracker.py      # Token追踪器测试
│   └── test_model_config_manager.py # 配置管理器测试
├── run.py                         # 启动脚本
├── requirements.txt               # 依赖列表
├── QUICK_START.md                # 快速启动指南
└── README.md                      # 本文档
```

### 技术栈

- **GUI框架**: Tkinter (Python内置)
- **LLM集成**: LangChain + OpenAI SDK
- **异步处理**: threading (后台线程)
- **数据持久化**: JSON文件
- **Python版本**: 3.8+

## 📚 功能详细说明

### 项目分析

**操作步骤：**
1. 点击 "📁 选择项目" 选择Java项目
2. 点击 "🔍 分析项目"
3. 等待分析完成（10-30秒）
4. 查看生成的 `overview.md`

**分析内容：**
- 项目框架及技术栈
- 学习路线图
- 项目代码索引

---

### 知识点学习

**操作步骤：**
1. 项目分析完成后，点击 "📚 下一知识点"
2. 系统生成知识点讲解
3. 保存到 `note/{topic}.md`
4. 自动切换到文档视图

**学习流程：**
```
概念解释 → 核心原理 → 项目实战 → 常见陷阱 → 进阶技巧
```

---

### AI问答

**操作步骤：**
1. 在学习知识点后，输入框自动启用
2. 输入问题并按Enter或点击 "💬 提问"
3. 等待AI回答（2-5秒）
4. 自动切换到问答记录视图

**示例问题：**
- "@SpringBootApplication注解的作用是什么？"
- "这个知识点在项目哪里用到了？"
- "能给我一个更简单的例子吗？"

---

### Token消耗监控

**查看方式：**
1. 切换到 "💰 Token消耗" Tab
2. 查看统计信息（总调用次数、总Token）
3. 查看详细记录表格

**记录内容：**
- 操作类型（分析/讲解/问答）
- 模型名称
- 输入/输出Token数
- 时间戳

**生成报告：**
点击 "生成Token报告" 按钮查看详细报告

---

### 模型配置管理

**快速切换：**
1. 在工具栏 "模型配置" 下拉框选择配置
2. 系统自动加载该配置的所有参数

**管理配置：**
1. 点击 "⚙️ 管理配置"
2. 左侧选择配置，右侧编辑
3. 支持：编辑、设为默认、删除、保存

## 💡 使用技巧

### 1. 分阶段使用不同模型

**推荐策略：**
```
项目分析: GPT-4 (质量优先)
知识点讲解: GPT-3.5-turbo (成本优化)
问答互动: GPT-3.5-turbo (速度快)
```

**操作方法：**
1. 创建两个配置：`GPT-4 分析` 和 `GPT-3.5 讲解`
2. 分析项目前选择 `GPT-4 分析`
3. 讲解知识点时切换到 `GPT-3.5 讲解`
4. **预计节省：60% API费用**

### 2. Token成本参考

| 操作 | Tokens | GPT-4成本 | GPT-3.5成本 |
|------|--------|-----------|-------------|
| 项目分析 | ~6000 | $0.18 | $0.012 |
| 知识点讲解 | ~5000 | $0.15 | $0.01 |
| 回答问题 | ~2000 | $0.06 | $0.004 |

### 3. 配置管理

- 设置默认配置，程序启动自动加载
- 定期查看使用统计，优化配置选择
- 删除不再使用的配置，保持列表整洁

## 🔧 故障排除

### 问题1: 导入错误

**错误信息：**
```
ModuleNotFoundError: No module named 'langchain'
```

**解决方案：**
```bash
pip install -r requirements.txt
```

---

### 问题2: API密钥无效

**错误信息：**
```
AuthenticationError: Invalid API key
```

**解决方案：**
1. 检查API密钥是否正确复制（无多余空格）
2. 确认账户余额充足
3. 尝试重新生成API密钥

---

### 问题3: 项目分析失败

**错误信息：**
```
RuntimeError: 项目分析失败: ...
```

**解决方案：**
1. 确认项目路径包含 `src/main/java`
2. 检查是否有 `pom.xml` 或 `build.gradle`
3. 查看 `overview.md` 中的详细错误信息

---

### 问题4: 界面冻结

**症状：**
- 点击按钮后界面无响应
- 无法拖动窗口

**解决方案：**
1. 等待30秒，可能是LLM响应慢
2. 如果持续无响应，强制关闭应用
3. 检查网络连接
4. 尝试降低模型复杂度（改用GPT-3.5）

---

### 问题5: Token记录未保存

**原因：**
- 项目路径未正确设置

**解决方案：**
确保创建orchestrator时传入project_path（GUI自动处理）

---

### 问题6: 配置未加载

**原因：**
- 配置文件损坏或格式错误

**解决方案：**
1. 删除 `config/model_configs.json`
2. 重启应用，自动创建默认配置
3. 重新添加配置

## 🛡️ 安全建议

### API Key保护
1. 优先使用环境变量存储API Key
2. 将 `config/model_configs.json` 加入 `.gitignore`
3. 定期轮换API Key

### 数据安全
- 项目代码不会上传到外部服务器（仅发送给LLM API）
- API密钥仅存储在内存和本地配置文件
- 建议使用环境变量而非硬编码
